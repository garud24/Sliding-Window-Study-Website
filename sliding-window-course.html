Theory

What is a Sliding Window?

A contiguous subarray or substring that moves through data. Instead of recomputing from scratch for every subarray — O(n²) — you slide the window: add the new rightmost element, drop the old leftmost. Result: O(n).

Fixed Fixed Size Window

  • Problem gives you an exact size k
  • Max/min/sum/average over every k elements
  • Permutation or anagram matching
  • Window moves exactly 1 step at a time

Variable Variable Size Window

  • "Longest/shortest subarray/substring"
  • "At most k distinct / zeros / flips"
  • Expand right freely, shrink left when invalid
  • Answer updates every iteration

Core Templates

Fixed Window O(n) 
# Build first window
window = sum(arr[:k])
best = window

for i in range(k, len(arr)):
    window += arr[i] - arr[i-k]
    best = max(best, window)
Variable Window O(n) 
left = 0
for right in range(len(arr)):
    # expand: add arr[right]
    while <window invalid>:
        # shrink: remove arr[left]
        left += 1
    ans = max(ans, right-left+1)
—

When to use sliding window

✓ Use it when

Problem mentions contiguous subarray/substring, max/min/count in a window, "at most k", "exactly k", longest/shortest.

✗ Don't use it when

Subsequences (not contiguous), elements can be reused, or you need all permutations/combinations (use recursion/backtracking).

Key insight

Every element enters the window once and leaves once → total work = 2n = O(n) even with the inner while loop.

Cheat Sheet

Interview Decision Tree

1.Is it a contiguous subarray/substring problem? → YES → Sliding Window
2.Is window size fixed? → Use fixed template
3.Is window size variable? → Expand right, shrink left while invalid
4.Need frequency count? → Add HashMap / Counter
5.Need window max/min? → Add Monotonic Deque
6.Exactly k something? → atMost(k) − atMost(k−1) trick

Key Tricks

Exactly k → atMost(k) − atMost(k−1)
Converts hard exact-count problems into two easy variable-window passes. Works for distinct chars, ones, specific values.
Monotonic deque for window max/min
Maintain deque of indices in decreasing order. Front = current max. Pop front if out of window. Pop back if smaller than new element.
Count all valid subarrays ending at right
When window is valid, ans += right − left + 1. This counts all valid subarrays ending at the current right pointer.
Write the VIOLATION condition
In your while loop: while distinct > k, not while valid. Shrink when constraint is BROKEN.
